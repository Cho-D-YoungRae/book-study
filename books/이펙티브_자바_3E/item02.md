# 아이템 2. 생성자에 매개변수가 많다면 빌더를 고려하라

- 정적 팩터리와 생성자의 제약: 선택적 매개변수가 많을 떄 적절히 대응하기 어렵다
- 대안
  - 점층적 생성자 패턴
    - 매개변수가 많아지면 클라이언트 코드를 작성하거나 읽기 어려움
  - 자바빈즈 패턴
    - 객체 하나를 만들려면 메서드를 여러 개 호출해야 하고, 객체가 완전히 생성되기 전까지는 일관성이 무너진 상태에 놓이게 됨
      - 매개변수들이 유효한지를 생성자에서만 확인하면 일관성을 유지할 수 있었는데, 그 장치가 완전히 사라진 것
    - 클래수를 불변으로 만들 수 없음
- **빌더 패턴**
  - 명명된 선택적 매개변수를 흉내 낸 것
  - 빌더 패턴은 계층적으로 설계된 클래스와 함께 쓰기에 좋음
